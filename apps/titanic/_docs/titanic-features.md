# 타이타닉(Titanic) 데이터셋 분석 및 피처 엔지니어링 가이드

## 1. 개요 (Overview)
타이타닉 데이터셋은 승객들의 정보를 바탕으로 생존 여부를 예측하는 머신러닝 입문용 데이터셋입니다.
- **train.csv**: 모델 학습용 데이터 (생존 여부인 'Ground Truth' 포함)
- **test.csv**: 모델 평가용 데이터 (생존 여부 미포함, 예측 대상)
- **gender_submission.csv**: 제출 양식 예시 (모든 여성 승객이 생존했다는 가정 하의 결과)

## 3. 상세 변수 노트
- **pclass**: 사회경제적 지위(SES)의 대리 지표 (1=상류층, 2=중산층, 3=하류층)
- **sibsp**: 
    - Sibling: 형제, 자매, 의붓형제, 의붓자매
    - Spouse: 남편, 아내 (애인 등은 제외)
- **parch**:
    - Parent: 어머니, 아버지
    - Child: 딸, 아들, 의붓딸, 의붓아들
    - 참고: 유모와 함께 여행한 어린이는 parch=0으로 처리됨.

---

## 4. 피처 엔지니어링(Feature Engineering) 아이디어
모델 성능 향상을 위해 다음과 같은 변수 생성을 고려해 보세요.

1. **가족 규모 생성**: `sibsp` + `parch` + 1(본인)을 합쳐 '가족 크기' 변수 생성
2. **나이 그룹화**: 나이대별(어린이, 청년, 중년, 노년)로 카테고리화
3. **요금 구간화**: `fare`가 너무 넓은 범위를 가지므로 로그 변환이나 구간(binning) 나누기
4. **객실 정보 활용**: `cabin`의 알파벳(구역)만 추출하여 객실 위치 특성 도출
5. **호칭 추출**: `Name` 컬럼(제공된 데이터에는 없으나 원본 존재 시)에서 'Mr', 'Miss', 'Mrs', 'Master' 등의 호칭을 추출하여 성별 및 사회적 위치 추정
